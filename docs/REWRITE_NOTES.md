# 原型审查与重构说明

## 原型主要问题

| 原型问题 | 风险 | FluxGate 处理方式 |
|---|---|---|
| 使用字符数组手工解析伪 XML 配置 | 越界、字段残留、错误难定位 | 重写为严格 INI 解析器，逐行校验范围和重复名称 |
| `open()` 结果用 `if (!fd)` 判断 | 文件描述符为 0 时误判，-1 时反而可能漏判 | 所有系统调用按 `< 0` 判断并抛出含 errno 的异常 |
| 大量 `assert` 处理运行时错误 | Release 构建失去保护，服务直接退出 | 区分配置错误、系统调用错误和连接级错误 |
| 后端连接预建并阻塞式 `connect/sleep` | 启动慢、事件循环被阻塞、故障恢复迟缓 | 每个客户端按需非阻塞连接，独立健康检查恢复节点 |
| 两个固定 2 KiB 裸缓冲区 | 容量过小，易触发停顿，手工内存管理 | 可配置固定容量 `ByteBuffer`，游标读写和原地压缩 |
| 同一连接对象被裸指针放入多个 map | 生命周期不清晰，容易重复释放或悬空 | `shared_ptr<ProxySession>` + 延迟删除 + RAII |
| 父子进程仅发送 1 字节繁忙度 | 活动连接数大于 255 时截断 | Worker 使用原子活动会话计数，主线程直接读取 |
| 进程数与后端数绑定 | 扩缩容和 CPU 利用率受后端数量限制 | Worker 数量与后端数量完全解耦 |
| 缺少健康检查 | 故障节点仍可能被选中 | 周期 TCP 探活、自动摘除和自动恢复 |
| 未完整处理短写、半关闭和背压 | 丢数据、忙循环或连接提前关闭 | 双向状态机、EAGAIN、shutdown 和事件兴趣动态调整 |
| 日志非线程安全 | 多线程输出交叉 | 带互斥保护的分级日志器 |
| 仅 Makefile，无测试 | 构建不可移植，修改难验证 | CMake、CTest、Sanitizer、集成和故障转移脚本 |

## 重构边界

FluxGate 没有对原类做机械式改名，而是重新定义了模块职责：

- `processpool` 被 Acceptor + Worker Reactor 替代；
- `mgr` 被 BackendPool、HealthChecker 和 Worker 替代；
- `conn` 被拥有完整双向状态的 ProxySession 替代；
- `fdwrapper` 被 UniqueFd 与 Net 工具替代；
- 配置、指标和测试被拆成独立模块。

因此新项目在代码组织、对象关系、并发策略和故障处理上均与原型不同。
