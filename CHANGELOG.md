# Changelog

## v1.2.0 - 2026-07-02

- 根据真实 ApacheBench 与 wrk 结果更新 README、压测、验证和简历文档。
- 记录 200 并发 60 秒、500 并发 300 秒持续压测结果与适用边界。
- 新增 `docs/CODE_GUIDE.md`，逐文件说明类、函数、数据结构和调用关系。
- 为全部 C++ 头文件补充类、函数、成员和资源所有权注释。
- 为核心实现补充非阻塞连接、epoll、背压、半关闭和延迟删除说明。
- 为 CMake、默认配置、测试脚本和 Python 后端补充用途与参数注释。
- 将压测结束时常见的 ECONNRESET/EPIPE 收尾事件降为 DEBUG，保留其他 Socket 错误为 WARN。
- 修正自动报告中代理吞吐高于直连时出现“负数损耗”的表述。
- 更新项目版本为 1.2.0，并整理可提交到 GitHub 的精选参考压测结果。

## v1.1.0 - 2026-07-02

- 增加完整压力测试脚本、Nginx/Python 测试后端和自动化 Markdown 报告。
- 增加 50、100、200 并发 keep-alive、短连接及单节点摘除压力场景。
- 修复业务连接单次失败立即摘除后端导致的级联拒绝风险。
- 健康检查增加连续失败和连续成功阈值，形成状态迟滞并降低抖动误判。
- 数据面连接失败改为只记录指标，节点状态统一由主动健康检查维护。
- 增加参考压测结果和复现文档。

## v1.0.0 - 2026-07-02

- 使用 CMake 和 C++11 创建全新工程结构。
- 重写为 Acceptor + 多 Worker Reactor 架构。
- 增加非阻塞 TCP 双向代理、短写处理、背压与半关闭。
- 增加加权最少连接负载均衡算法。
- 增加后端主动健康检查、自动摘除和恢复。
- 增加连接超时、空闲超时、优雅退出和运行指标。
- 增加 RAII 文件描述符与后端租约管理。
- 增加配置解析和严格参数校验。
- 增加 CTest 单元测试、HTTP 集成测试和故障转移测试。
- 增加 AddressSanitizer/UndefinedBehaviorSanitizer 构建选项。
- 增加架构、重构、部署、简历和面试文档。
